# Database — SQL Server

The structured metadata produced by the [ETL pipeline](../etl) is stored in a
SQL Server database named **`ETL_VENU`**. This folder contains the schema and
the queries used to populate the Power BI dashboard.

## Contents

| File | Purpose |
|------|---------|
| `schema.sql` | Creates the `ETL_VENU` database and the five plant tables. |
| `queries/select_*.sql` | Simple preview queries — one per destination table. |
| `queries/analysis_queries.sql` | Aggregations that back the Power BI charts. |

## Table layout

The SSIS package routes rows by plant type into five tables, all sharing the
same columns:

`dbo.Tomato`, `dbo.Peach`, `dbo.Potato`, `dbo.Corn`, `dbo.Other_Plants`

| Column | Description |
|--------|-------------|
| `plant_type` | Crop name (e.g. Tomato, Corn (maize)). |
| `disease_name` | Disease label, or "healthy". |
| `class_name` | Original `Plant___Disease` folder label. |
| `disease_severity` | Healthy / Mild / Moderate / Severe. |
| `Copy of *` | Type-converted copies created by the Data Conversion step. |
| `NUmb Label` | Severity encoded as an integer (Healthy=0, Mild=1, Moderate=2, Severe=3). |
| `Plant_cat` | Plant category — Root / Grain / Fruit / Vegetable / etc. |
| `Severity_risk_level` | "Low" (Healthy/Mild) or "High" (Moderate/Severe). |
| `is_fruit_crop` | 1 if the crop is a fruit crop, otherwise 0. |

## Setup

1. Open **SQL Server Management Studio (SSMS)** and connect to your instance.
2. Run `schema.sql` to create the database and tables.
3. Run the [SSIS package](../etl) to load data into the tables.
4. Run `queries/analysis_queries.sql` to create the `vw_AllPlants` view and
   verify the aggregations.

> The `select_*.sql` files were generated by SSMS as `SELECT TOP 1000`
> previews and are kept for quick inspection of each table.
